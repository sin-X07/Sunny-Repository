# 1. 通讯录管理系统 (Contect Management System) 2026.3

一个基于 C++ 开发的轻量级命令行通讯录管理系统，支持联系人信息的增删改查及本地文件的持久化存储。

## 功能特性

* **联系人管理 (CRUD)**
    * **添加联系人**：记录姓名、性别、年龄、地址及手机号。
    * **删除联系人**：根据姓名查找并删除指定联系人。
    * **修改联系人**：支持对已有联系人的各项信息进行更新。
    * **查找联系人**：通过姓名快速检索联系人详细信息。
    * **显示联系人**：列表化展示当前系统中的所有联系人。
* **动态扩容**：内置基于动态数组的内存管理，当联系人数量达到上限时自动触发扩容，确保数据正常写入。
* **数据持久化**：
    * 程序启动时自动读取本地 `addressbook.txt` 文件加载历史数据（若文件不存在则自动创建）。
    * 正常退出程序时自动将当前数据保存回本地文件。
* **界面交互**：提供直观的清屏与暂停机制，保证命令行界面的整洁与操作连贯性。

## 项目结构

```
ContectManagmentSystem
├── Person.h          # 核心数据结构（member 结构体）与 AddressManager 类声明
├── AddressBooks.cpp  # 业务逻辑实现（文件 I/O、内存扩容、各功能具体实现）
├── main.cpp          # 程序入口，负责主循环与菜单事件分发
└── addressbook.txt   # 本地数据存储文件（程序运行后自动生成）
```


# 2. Concurrent KV-Storage Engine (Skip List) 2026.5

一个基于 C++14 实现的单机键值（KV）存储引擎。内存核心数据结构采用跳表（Skip List），并结合了日志先行（WAL）与快照（Snapshot）技术以提供数据持久化能力，同时使用读写锁实现多线程并发控制。

---

## 功能特性

* **内存结构**：使用泛型模板实现的 Skip List，支持动态插入、删除、修改和查找操作，各项操作的时间复杂度为 $O(\log n)$。
* **数据持久化 (WAL & Snapshot)**：
    * **WAL (Write-Ahead Logging)**：在修改内存数据前，先将操作指令（`SET` 或 `DEL`）顺序写入日志文件并执行 `flush()`，保障断电时的数据安全。
    * **Snapshot (全量快照)**：支持将当前内存中的有效数据全量导出至快照文件。快照完成后会清空当前的 WAL 日志，以释放磁盘空间并控制下次启动时的重放耗时。
* **断电恢复机制**：系统冷启动时，先通过 `load_file()` 加载最近一次的快照文件，随后通过 `replay_wal()` 按顺序重放快照之后的增量日志。在重放与加载过程中关闭了写日志逻辑，防止出现重复记录。
* **并发控制 (Thread-Safety)**：使用 C++ 标准库的 `std::shared_mutex`（读写锁）进行并发管理。
    * 对于 `search_element` 和 `display_list` 等只读操作，使用 `std::shared_lock`（共享读锁），允许多个线程并行读取。
    * 对于 `insert_element`、`delete_element` 和 `dump_file` 等修改结构的操作，使用 `std::unique_lock`（独占写锁），确保线程安全。
* **生命周期与锁分离**：`load_file` 和 `replay_wal` 函数仅在数据库初始化的单线程冷启动阶段调用，不进行上锁操作，以规避潜在的死锁隐患并提升启动效率。
